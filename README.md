# Audit Client

Клиентская библиотека для автоматического сбора и отправки аудиторских событий в системе централизованного аудита.

## Описание проекта

`audit-client` — Spring Boot starter, предназначенный для подключения к прикладным Java-сервисам без существенной модификации бизнес-логики.

Библиотека обеспечивает:

* автоматический перехват методов, помеченных аннотацией `@Audit`;
* формирование структурированного аудиторского события;
* извлечение контекста выполнения;
* отправку события в центральный сервис аудита;
* поддержку асинхронной передачи через Apache Kafka.

Проект разработан в рамках выпускной квалификационной работы и является клиентской частью системы централизованного аудита и анализа событий.

---

# Основные возможности

* декларативное подключение аудита через аннотации;
* перехват бизнес-методов с помощью Spring AOP;
* сбор параметров аудиторского события;
* определение пользователя, роли, IP-адреса и идентификатора запроса;
* фиксация времени выполнения метода;
* формирование результата выполнения операции;
* отправка событий через реализацию `AuditSender`;
* поддержка режимов с Kafka и без Kafka.

---

# Назначение библиотеки

Библиотека предназначена для интеграции в прикладные Java-приложения, где требуется:

* вести аудит действий пользователей;
* централизованно передавать события в аудиторский сервис;
* минимизировать объём изменений в существующем коде;
* обеспечить единообразный формат событий;
* использовать один механизм аудита в нескольких сервисах.

```text id="l3s9k1"
AuditService

    ↓
Kafka / NoOp / другая реализация
```

## Компоненты библиотеки

### `@Audit`

Аннотация для указания методов, подлежащих аудиту.

### `AuditAspect`

Spring AOP аспект, который перехватывает вызовы методов и формирует событие.

### `AuditService`

Сервис, который выполняет валидацию и передаёт событие реализации `AuditSender`.

### `AuditSender`

Интерфейс отправки события. Позволяет использовать разные способы доставки.

### `KafkaAuditSender`

Реализация для отправки событий в Apache Kafka.

### `NoOpAuditSender`

Заглушка, используемая в окружениях без брокера сообщений.

# Структура аудиторского события

Событие может включать:

* тип операции;
* имя сервиса;
* класс и метод;
* идентификатор пользователя;
* роль пользователя;
* IP-адрес;
* идентификатор запроса;
* состояние объекта до и после изменения;
* статус выполнения;
* временную метку;
* длительность выполнения.

# Использование

## 1. Подключение зависимости

Добавьте библиотеку в проект как dependency.

## 2. Включение аннотации

```java id="p1k8zd"
@Audit(action = "TRANSFER_FUNDS")
public void transferMoney(...) {
    ...
}
```

## 3. Настройка параметров

Укажите параметры подключения в `application.properties` или `application.yml`.

---

# Пример применения

```java id="t4n8mp"
@Service
public class PaymentService {

    @Audit(action = "PAYMENT_CREATE")
    public void createPayment(PaymentRequest request) {
        // business logic
    }
}
```

После вызова метода библиотека автоматически:

1. перехватывает вызов;
2. формирует событие;
3. добавляет контекст выполнения;
4. отправляет событие в `audit-service`.
